# 策划变更审阅工作区

把会议纪要或零散灵感视为对现有策划体系的一组增量变更，先完成证据对齐、冲突检查和影响面审阅，再跟踪结论是否真正进入策划案。

## 核心流程

1. 在“知识库”指定本地策划文件夹，并把资料标记为“核心”“参考”或“忽略”。
2. 在“审阅”粘贴会议纪要或灵感，填写当前目标。
3. 系统区分决策、提案、问题、行动、风险和信息。
4. 每个讨论点只引用检索到的真实知识片段，并保留文件路径与行号。
5. 查看新增、修改、冲突、缺口，以及直接影响、下游影响和相关讨论点。
6. 人工选择“纳入变更”“需澄清”或“暂不纳入”。
7. 生成变更包，冻结全部“纳入变更”“需澄清”“暂不纳入”和待审结论。
8. 手动修改本地策划案后运行“变更验证”，系统自动重扫文件夹并比较修改前版本、会议结论与修改后版本。
9. 逐条确认“已落实”“保持不纳入”“仍需澄清”“意外写入”等结果；后续改动可以继续生成 V2、V3 验证记录。

## 启动

```powershell
npm install
npm run dev
```

前端默认地址：`http://127.0.0.1:5173`
接口默认地址：`http://127.0.0.1:8787`

## 数据

- 模型设置保存在项目根目录的 `.env`。
- 会议、审阅项和变更包保存在 `data/store.json`。
- 每次分析的修改前版本和每轮验证版本保存在 `data/snapshots`，内容按 SHA-256 去重。
- 上传文件保存在 `data/uploads`。
- 手动导出的 Markdown 保存在 `data/outputs`。
- 本地知识库始终只读扫描，不会自动修改源文件。

未配置模型时会使用保守的本地规则；配置 OpenAI 兼容接口后，分析采用“讨论点提取 -> 候选知识检索 -> 证据约束审阅”，验证采用“修改前证据 -> 人工决定 -> 修改后证据”的三方核对流程。
